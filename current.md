## Section 3: Functional Requirements

| ID | Priority | Description |
|----|----------|-------------|
| FR-01 | Must | System shall provide interactive questionnaire to collect project details including type (digitalization, climate protection, infrastructure), scope, budget, and municipality information. |
| FR-02 | Must | System shall search funding databases (FÃ¶rderdatenbank, KfW, BAFA, EU sources) using semantic matching based on questionnaire inputs. |

| FR-04 | Must | System shall categorize projects into predefined packages (e.g., digitalization small/medium/large, infrastructure, climate measures). |
| FR-05 | Must | System shall direct users to association membership application form after displaying funding opportunities. |
| FR-06 | Must | System shall capture and store municipality contact information and identified funding interests for follow-up consultation. |
| FR-07 | Must | System shall provide informational content about Klimafolgenschutzverein mission and services. |
| FR-08 | Deferred | System shall accept document uploads (cost estimates, planning documents) for enhanced funding search precision. |
| FR-09 | Deferred | System shall enable direct submission of funding applications to BAFA/KfW portals on behalf of municipalities. |
| FR-10 | Deferred | System shall integrate with EE-Mann (energy efficiency expert) workflow for projects requiring expert validation. |
